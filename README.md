# OBD2019
**Структура базы данных системы управления проектами (БД СУП).**

База данных, в общем случае, осуществляет интеграцию данных, создавая динамически (в реальном масштабе времени) обновляемую модель предметной области (проекта) в виде единого для всех задач блока данных. Упрощенно говоря, структура БД – это содержавшиеся в БД таблицы и их связи.  Структура БД  позволяет поддерживать логику каждой конкретной задачи, выполняемой над хранимыми данными и поддерживает возможность организации данных с точки зрения представления пользователя.
Структура БД СУП является ключевым рабочим продуктом, необходимым при работе с проектом.  Она представляет собой иерархический перечень рабочих действий, необходимых для создания проекта и работы с ним. Структура образует своего рода каркас, на основе которого проектируется все компоненты проекта.  
Таким образом, БД СУП предназначена для создания иерархических связей между элементами проекта и рабочими действиями (элементами процесса) и  содержит следующие элементы (типовая структура): 
1.	Исходные данные, базовые процессы и граничные условия проекта: ресурсы (объекты, которые необходимы в процессной деятельности (персонал, оборудование и др.); назначения (связь определенной задачи и ресурсов, необходимых для ее выполнения. Она позволяет рассчитать объем времени, затраченный назначенным  на проект, его стоимость для проекта, производить динамическое перераспределение ресурсов); время, стоимость, объем работ;  задачи (работа, осуществляемая в рамках проекта для достижения определенного результата, которая должна быть завершена в течение некоторого периода времени); фазы (собрание связанных задач, в процессе выполнения которых реализуется производство определенного продукта. Если для достижения результатов задачи нужно выполнить только ее, то для достижения результата фазы нужно выполнить группу других задач. Фазы также служат для удобства контроля работ);

2.	Установление связей между данными, ограничений, фильтров и позволяющие структурировать и совместно использовать данные для описания процессов - формирования (планирования) проектов различного масштаба (проектов, мультипроектов, подпроектов). Создание плана проекта заключается в описании его ключевых характеристик (длительность, рабочее время, методика планирования); задач проекта и связей между ними (в MS Project поддерживается 4 типа связей: окончание-начало, начало-начало, окончание-окончание, начало-окончание); доступных ресурсов и определение взаимосвязи между ними с помощью назначений: ограничений (привязывание задач к определенным датам); крайних сроков (предельная дата исполнения задачи); стоимости; суммарной задачи проекта объединяющей все проектные активности).
К типовым наборам свойств задачи проекта относятся: длительность задачи; трудозатраты; назначенные ресурсы; ограничения; крайние сроки; приоритет (важность ее исполнения для проекта).
К типовым наборам свойств ресурсов проекта относятся: стоимость; загрузка; доступность; назначения; приоритет. 
Для работы с БД СУП в MS Project используются интерфейс и шаблоны. 


**2. Пользовательский интерфейс (виды представлений информации о проекте, панели инструментов).**

>2.1. Виды представлений информации о проекте.

Представление – это способ  отображения проектных данных для просмотра и редактирования. Micrisoft Progect содержит следующие виды представлений информации о проектах (составляющих представления): таблицы (листы), диаграммы и формы.  Файл проекта содержит огромное количество данных, и просматривать их одновременно невозможно. В представлении на экране отображается ограниченный набор нужной информации о проекте, что облегчает ее просмотр и редактирование. Представление имеет гибкую структуру и может состоять как из отдельной таблицы, диаграммы или формы, так и из их комбинации.
Представление может быть одно или как комбинация нескольких представлений (диаграмма Ганта, лист ресурсов и т. д.) и, если применимо, таблицы или фильтра. Представления позволяют работать с информацией в разных форматах.
Менеджмент процесса (понимаемого как последовательность шагов, т.е. ряда действий и операций, направленного на достижение определенной цели) начинается с проектного (организационного) подхода.  Внешний интерфейс процесса охватывает всю информацию, которая может использоваться для его определения.  Для сложившейся организации при этом обеспечивается представление основ знаний предыдущего проекта, методов измерения продукта и др. параметров. Для новой организации или организации, которая является "новой" в рамках процесса, интерфейс представляет области, которые надо "посетить" с целью сбора информации и данных, необходимых для запуска проекта.
Таблицы – это одно из основных средств представления проектных данных в MS PROJECT. 
Типы таблиц MS Project
В проектном файле все данные хранятся в виде двух таблиц, одна из которых! держит информацию о задачах, а вторая — о ресурсах проекта, то есть задействованных в выполнении задач людях и материальных ценностях. Эти две "внутренние" таблицы составляют основу файла проекта и состоят из множества полей, большинство из которых создано "про запас" и обычно не используются. Некоторые из них могут быть задействованы при программировании, в тонких настройках, функциях анализа проекта и т.д., но не нужны при стандартном использовании MS PROJECT. Во время работы над проектом, как правило, требуются вполне определенные срезы информации. Например, данные о стоимости проекта, загруженности или иных ресурсов, сроках окончания работ по текущим задачам и т.п. То есть в каждый конкретный момент времени требуется иметь перед глазами то необходимую информацию. Это позволяет сконцентрироваться на ней и эффективно работать.
Файл проекта состоит из двух больших "внутренних" таблиц с множеством столбцов, из которых только некоторые отображаются в таблицах, включенных в представления.  Работа в MS Project, происходит в основном только с маленькими "внешними" таблицами. В MS Project заложен набор предопределенных таблиц, предназначен для работы с небольшим объемом проектной информации (табл. 1 и 2 Приложения) из этих таблиц содержит лишь несколько полей одной из "внутренних" таблиц проектного файла. Соответственно, "внешние" таблицы (как и "внутренние) содержат информацию либо о задачах, либо о ресурсах.
Таким образом, "внутри" MS Project есть таблица ресурсов и таблица задач, а "снаружи", в интерфейсе, есть ряд таблиц, являющихся набором полей либо из  одной, либо из другой "внутренней" таблицы.
Диаграммы.
Диаграмма является графическим средством отображения содержащейся в проектном файле информации. Из диаграммы можно получить визуальное представление о последовательности задач, их относительной длительности, о длительности проекта в целом. В MS PROJECT диаграммы являются не только средством просмотра проектной информации, но с их помощью можно вводить новые данные и редактировать существующие. 
Диаграмма Ганта представляет собой средство для представления длительности и последовательности задач в проекте.  Так, все элементы диаграммы Ганта – основного средства визуализации плана проекта (время – задачи) в программе являются настраиваемыми отрезками, каждый из которых может состоять из 3 элементов: точки начала, точки окончания и промежуточной части (при этом любой из элементов может отсутствовать). При стандартной настройке отрезки, обозначающие фазы, состоят из всех трех элементов. Отрезки, обозначающие задачи, состоят только из промежуточной части, а завершающие задачи – только из начальной  точки. 
На диаграмме Ганта  рядом с отрезками может отображаться дополнительная информация (например, на рисунке рядом с задачами, отображаются названия задействованных в них ресурсов и их загрузка при выполнении задачи). Состав этой информации определяется  настройками диаграммы, в MS PROJECT входит несколько заранее настроенных версий диаграммы Ганта, на которых рядом с отрезками отображается различная информация (Табл. 3 Приложения).
MS PROJECT, кроме диаграммы Ганта, содержит 3 сетевых графика (или блок-схемы): собственно Network Digram (сетевой график), Discriptiv Network Digram (сетевой график с описанием), и  Relationship Digram (схема данных или диаграмма зависимостей). Отличие сетевого графика с описанием от стандартного сетевого графика заключается только в повышенной информативности блоков В них обоих можно сразу просматривать информацию обо всем плане проекта, а на схеме данных – только об одной задаче этого плана; она предназначается для анализа взаимосвязей между задачами.
Одна из основных диаграмм при работе с MS PROJECT – Calendar (Календарь), с помощью которого представляется план работ в виде традиционного календаря.  Диаграмма Календарь отображает информацию о плане проекта в виде таблицы с семью или пятью колонками, соответствующими дням недели, и множеством рядов, соответствующих неделям. Задачи на календаре обозначаются отрезками, которые начинаются в день начала работ над задачей и заканчиваются в день их окончания.
Диаграмма Resourse Graph (График ресурсов) содержит информацию о различных аспектах участия ресурсов в проекте: выполняемой работе, процентной загрузке, возможности выполнять другие задачи помимо тех, на которые они уже выделены, и стоимости. С помощью этой диаграммы можно сравнивать участие в проекте нескольких ресурсов или ресурса и группы ресурсов, что делает ее мощным аналитическим инструментом.  Группа ресурсов отбирается путем фильтрации, по умолчанию в качестве группы выступают все ресурсы проекта. Изменив условие фильтрации, можно изменить и состав группы. Данные можно просматривать как для группы в целом, так и для любого отдельного ресурса внутри группы. 
Диаграммы использования задач и ресурсов в большей степени, чем остальные, применяются для ввода повременных данных в файл проекта. 
Диаграмма Task Usage (Использование задач) предназначена для анализа  участия ресурсов в проектных работах. Сама диаграмма представляет собой таблицу, в строках которой размещена информация о фазах, задачах и ресурсах. Внутри строки данные размещены в ячейках в соответствии с датами, к которым они относятся. Таким образом диаграмма позволяет получить разнообразные данные о работе (объем работы, стоимость, загрузка ресурсов и пр.) за произвольный временной период для проекта в целом, любой из его фаз, задач или задействованного в выполнении задач ресурса. Диаграмма Resource Usage (Использование ресурсов) предназначена для анализа за любой период времени как общей загрузки ресурса в проектных работах, так и из каких задач она складывается.
Формы.
Форма — это средство отображения и редактирования проектных данных в Project, которое может быть включено в состав представления. С помощью форм можно просматривать информацию только об одном объекте (ресурсе или задаче), но при этом можно редактировать почти все его свойства.
В MS PROJECT входят пять форм. С помощью первых двух можно просматривать и редактировать информацию о ресурсах, а трех последующих - о задачах: Resoure  Form (Форма ресурсов); Resoure Name Form (Форма названий ресурсов); Task Form  (Форма задач); Task Name Form (Форма названий задач); Task Details Form (Форма сведений о задачах).

>2.2. Создание новых представлений.

Чтобы создать новое представление, в диалоговом окне со списком представлений  нужно щелкнуть по кнопке New (Создать). В открывшемся окне  определяется тип создаваемого представления: Single view (Одиночное представление) или Combination view (Комбинированное представление). После выбора типа нового представления открывается диалоговое окно определения его свойств. В зависимости от того, какое представление создается, будет открыто диалоговое окно определения свойств одиночного или комбинированного представления.
В диалоговом окне определения свойств одиночного представления, в поле Name (Имя) указывается название нового представления. Это название затем появится в списке всех представлений и в меню, если установить флажок Show in menu (Показывать в меню). Раскрывающийся список Screen (Экран) позволяет выбрать диаграмму или форму, которая ляжет в основу представления. Перечень доступных вариантов ограничивается рассмотренными диаграммами  и формами. От пункта, выбранного в списке Screen (Экран), зависит, какие параметры будет настроить для создаваемого представления. Список Table (Таблица) доступен только в случае, если выбранный экран содержит таблицу. В этом списке можно выбрать таблицу, которая будет отображаться при загрузке представления. Список Group (Группа) также доступен только в том случае, если диаграмма допускает группировку данных. Тогда можно выбрать в списке группу, которая будет использоваться при группировке. Условия фильтрации данных можно определить для любой диаграммы с помощью  списка Filter (Фильтр), в котором перечислены существующие фильтры, предназначенные для отбора отображаемых в представлении данных. Для того чтобы использовать фильтрацию в режиме выделения, предназначен флажок Highlight filter (Выделять результаты фильтра). Если этот флажок установлен, любые фильтры, которые в дальнейшем будут применяться к данному представлению, также будут работать только в режиме выделения.
Диалоговое окно определения свойств комбинированного представления,  устроено проще аналогичного окна для одиночного представления. В поле Name (Имя) следует ввести название нового представления, а для того, чтобы оно отображалось в меню и на панели представлений, нужно установить флажок Show in menu (Показывать в меню). В раскрывающихся списках Тор (вверху) и Bottom (внизу) выбираются представления, которые будут отображаться в верхней и нижней частях нового представления. При этом список доступных представлений состоит не только из стандартных диаграмм (как в диалоговом окне создания одиночного представления), а из набора существующих одиночных представлений, в том числе и настроенных пользователем.
После того, как создание представления завершено, оно появляется в списке доступных представлений и его можно вызывать либо из списка, либо (если установлен соответствующий флажок) из меню или панели представлений. 
>2.3. Создание нового представления на основе существующего.

Часто удобнее создавать новое представление не "с нуля", а на базе уже существующего. Для этого щелчком по кнопке Сору (Копировать) в диалоговом  окне со списком представлений  выполняется копирование выделенного в  списке представления. После щелчка по кнопке Сору (Копировать) открывается диалоговое окно, похожее на окно создания представления. В этом окне к названию исходного представления добавляется Copy of (Копия), а на месте раскрывающегося списка Screen (Экран) располагается информационное поле, не дающее возможности сменить диаграмму или форму, являющуюся базовой для скопированного представления. После щелчка по кнопке ОК представление с новым названием сохраняется.
